# 自动化繁琐任务 - 详细说明

## 更新日期
2025-01-XX

## 更新内容

### 1. 四大任务类别

#### 🔍 代码质量维护
- **修复 Lint 错误**：自动修复 ESLint 和 TypeScript 错误
- **统一代码风格**：按 Airbnb 规范格式化所有文件
- **删除未使用代码**：清理未使用的导入和变量
- **性能优化**：优化 React 组件渲染性能

#### 📝 Git 工作流自动化
- **解决冲突**：智能处理 merge conflicts
- **生成提交**：创建描述性的 commit 信息
- **创建 PR**：生成包含详细描述的 Pull Request
- **发布说明**：根据 commits 生成 CHANGELOG

#### 📦 依赖包管理
- **升级依赖**：升级到最新兼容版本
- **安全审计**：检查并修复安全漏洞
- **清理未使用**：移除未使用的依赖包
- **迁移工具**：在不同构建工具间迁移（如 Webpack → Vite）

#### 📚 文档与测试
- **生成文档**：为 API 生成 JSDoc 和 Swagger 文档
- **编写测试**：为函数自动编写单元测试
- **提高覆盖率**：提升测试覆盖率到 90%+
- **README 生成**：根据代码生成完整的项目文档

### 2. 实际使用场景

#### 场景 1：准备发布
```bash
> claude
1. 运行所有测试并修复失败的测试
2. 更新 package.json 版本号
3. 生成 CHANGELOG
4. 创建 git tag
5. 推送到远程仓库
```

#### 场景 2：代码审查准备
```bash
> claude
- 格式化所有修改的文件
- 修复所有 lint 错误和警告
- 确保测试覆盖率 > 80%
- 为我的更改生成 PR 描述
```

### 6. 使用建议

#### 批量任务处理
可以一次性提交多个任务：
```bash
> claude
1. 修复所有 lint 错误
2. 运行测试并修复失败的测试
3. 更新文档
4. 创建 PR
```

#### 上下文记忆
使用 `#` 快捷方式添加项目上下文：
```bash
# 项目使用 TypeScript + React + Vite
# 测试框架是 Vitest
# 遵循 Airbnb 代码风格
```

#### 管道组合
将多个命令串联使用：
```bash
cat package.json | claude -p "升级所有依赖到最新版本"
```

### 7. 技术实现

#### DaisyUI 组件使用
- `card` - 任务类别卡片
- `border-l-4` - 左侧彩色边框
- `alert` - 实际场景示例框
- `stats` - 效率对比统计

#### Tailwind CSS 类
- `grid grid-cols-1 md:grid-cols-2` - 响应式网格
- `space-y-3` - 垂直间距
- `text-base-content/80` - 半透明文字
- `flex items-start gap-2` - 对齐和间距

#### SVG 图标
- 绿色对勾表示任务列表项
- 使用 `stroke-success` 颜色类
- 固定宽度和高度保持一致性
